Introducción

¿Qué es JavaScript y cuáles son sus principales usos en la web?

JavaScript es un lenguaje de programación que se usa sobre todo en las páginas web. Lo que tiene de interesante es que permite que un sitio no sea solo texto e imágenes estáticas, sino que tenga movimiento, interactividad y respuestas inmediatas sin necesidad de recargar todo.

¿En qué año y con qué propósito se creó JavaScript?

JavaScript fue creado en 1995 por Brendan Eich, El propósito era darle más dinamismo a las páginas web, Con JavaScript se buscaba validar formularios o mostrar cambios en la pantalla sin necesidad de recargarla por completo.

¿Por qué JavaScript no es lo mismo que Java?

Aunque los nombres son parecidos, JavaScript y Java son lenguajes de programación DIFERENTES. Java es un lenguaje más complejo y se usa para aplicaciones grandes, mientras que JavaScript es más sencillo y se usa principalmente en la web para hacer las páginas interactivas.

¿Cómo se complementa JavaScript con HTML y CSS?

HTML es el lenguaje que estructura el contenido de una página web, como los textos, imágenes y enlaces.
CSS es el lenguaje que se encarga del diseño y la apariencia, como los colores, fuentes y disposición de los elementos. JavaScript complementa a HTML y CSS al añadir interactividad y dinamismo, permitiendo que los usuarios puedan interactuar con la página de manera más fluida y atractiva.

Fundamentos del lenguaje

¿Cuál es la sintaxis básica de JavaScript y cómo se escriben comentarios?

La sintaxis básica de JavaScript se parece mucho a la de otros lenguajes de programación: se escriben instrucciones línea por línea y casi siempre terminan con un punto y coma ;.

Por ejemplo:

Si quiero mostrar un mensaje en la consola, usaría console.log("Hola");.
También se usan variables para guardar datos, que se pueden declarar con let, const o var.
Las llaves { } sirven para agrupar bloques de código, como en funciones o ciclos, y los paréntesis ( ) se usan al llamar funciones o en condiciones.
En cuanto a los comentarios, se escriben de dos formas: si es un comentario corto en una sola línea se usa // y para más líneas /* */

¿Qué diferencias hay entre var, let y const?

Existen tres formas principales de declarar variables: var, let y const, y aunque parecen parecidas, tienen diferencias importantes.

Var:

Es la forma más antigua de declarar variables. Tiene un alcance global o de función, lo que puede causar problemas si no se usa con cuidado, ya que una variable var puede ser redeclarada y actualizada en cualquier parte del código.

let:

Es una forma más moderna y recomendada de declarar variables. Tiene un alcance de bloque, lo que significa que solo existe dentro del bloque donde se declara (como dentro de un if o un for). Además, no permite redeclarar la misma variable en el mismo alcance, lo que ayuda a evitar errores.

Const:

También tiene un alcance de bloque, pero se usa para declarar constantes, es decir, valores que no deben cambiar una vez asignados. Al igual que let, no permite redeclarar la misma variable en el mismo alcance. Sin embargo, si la constante es un objeto o un array, sus propiedades o elementos sí pueden modificarse.

En resumen

var: antigua, menos segura, alcance de función.
let: se usa para variables que cambian, alcance de bloque.
const: para valores que no cambian, alcance de bloque.

¿Cuáles son los tipos de datos primitivos en JavaScript?

Los tipos de datos primitivos en JavaScript son los bloques básicos con los que se trabaja. Aquí te los explico:

Número (Number):

Representa tanto números enteros como decimales. Ejemplo: 42, 3.14

Cadena de texto (String):

Es una secuencia de caracteres encerrada entre comillas simples (' ') o dobles (" "). Ejemplo: "Hola, mundo!"

Booleano (Boolean):

Solo puede tener dos valores: true (verdadero) o false (falso). Se usa para representar condiciones. Ejemplo: true, false

Indefinido (Undefined):

Indica que una variable ha sido declarada pero no se le ha asignado ningún valor. Ejemplo: let x; // x es undefined

Nulo (Null):

Representa la ausencia intencional de cualquier valor. Es un valor que indica que no hay nada. Ejemplo: let y = null;

Símbolo (Symbol):

Es un tipo de dato único e inmutable que se usa para identificar propiedades de objetos de manera única. Ejemplo: let sym = Symbol('descripcion');

BigInt:

Es un tipo de dato que permite representar números enteros muy grandes, más allá del límite de los números normales en JavaScript. Se crea añadiendo una 'n' al final del número. Ejemplo: let bigIntNum = 123456789012345678901234567890n;

¿Qué diferencia hay entre un array y un objeto en JavaScript?

Un array y un objeto son dos estructuras de datos diferentes en JavaScript, cada una con sus propias características y usos.

Array:

Un array es una lista ordenada de elementos, donde cada elemento tiene un índice numérico que empieza en 0. Los arrays son ideales para almacenar colecciones de datos similares, como una lista de nombres o números. Se accede a los elementos del array usando su índice. Ejemplo: let frutas = ["manzana", "banana", "cereza"]; console.log(frutas[0]); // "manzana"

Objeto:

Un objeto es una colección de pares clave-valor, donde cada clave es una cadena (string) que identifica un valor asociado. Los objetos son útiles para representar entidades con múltiples propiedades, como un usuario con nombre, edad y correo electrónico. Se accede a las propiedades del objeto usando la clave. Ejemplo: let persona = {nombre: "Juan", edad: 30, correo: "

¿Qué son y cómo se usan los operadores aritméticos, lógicos y relacionales?

Operadores Aritméticos

Se usan para hacer cálculos matemáticos. Ejemplos:

let a = 10;
let b = 3;
console.log(a + b);
console.log(a - b);
console.log(a * b);
console.log(a / b);
console.log(a % b);
console.log(a ** b);

Estos operadores se usan para realizar operaciones matemáticas básicas como suma (+), resta (-), multiplicación (*), división (/), módulo (%) y exponenciación (**).

Operadores Lógicos

Se usan para combinar o invertir valores booleanos (true o false). Ejemplos:

let p = true;
let q = false;
console.log(p && q);
console.log(p || q);
console.log(!p);

Operadores Relacionales

Se usan para combinar o invertir valores booleanos (true o false). Ejemplos:

let x = 5;
let y = 8;
console.log(x > y);
console.log(x < y);
console.log(x >= 5);
console.log(x <= 4);
console.log(x == "5");
console.log(x === "5");
console.log(x != y);

Estos operadores se usan para comparar dos valores y devuelven un valor booleano (true o false). Incluyen mayor que (>), menor que (<), mayor o igual que (>=), menor o igual que (<=), igualdad (== y ===) y desigualdad (!= y !==).

Control de flujo

¿Cómo funcionan las sentencias condicionales if, else y switch?

if

Ejecuta un bloque de código si la condición es verdadera.

let edad = 18;
if (edad >= 18) {
console.log("Eres mayor de edad");
}

if else

Si la condición del if no se cumple, se ejecuta el bloque del else.

let hora = 10;
if (hora < 12) {
console.log("hello, how are you");
} else {
console.log("fine, thank you");
}

if--else if--else

Si la condición del if no se cumple, se ejecuta el bloque del else. Y permite comprobar múltiples condiciones.

let nota = 85;
if (nota >= 90) {
console.log("Excelente");
} else if (nota >= 70) {
console.log("Aprobado");
} else {
console.log("Reprobado");
}

Switch

Se usa cuando hay muchas opciones posibles para una sola variable.

let respuesta = 2;
switch (respuesta) {
case 1:
console.log("Si");
break;
case 2:
console.log("No");
break;
}

¿Cuáles son las diferencias entre los bucles for, while y do...while?

En JavaScript, los bucles (ciclos) sirven para repetir un bloque de código varias veces. Las diferencias principales entre for, while y do...while están en cómo controlan la condición.

for

Se usa cuando sabemos cuántas veces queremos repetir algo. Tiene 3 partes: inicialización, condición y actualización.

for (let i = 1; i <= 5; i++) {
console.log("Número: " + i);
}

while

Se usa cuando no sabemos cuántas veces se repetirá, pero depende de una condición. Primero verifica la condición, luego ejecuta.

let i = 1;
while (i <= 5) {
console.log("Número: " + i);
i++;
}

do while

Parecido a while, pero siempre ejecuta al menos una vez el bloque, porque la condición se revisa al final.

let i = 6;
do {
console.log("Número: " + i);
i++;
} while (i <= 5);

¿Para qué sirven las palabras clave break y continue?

Las palabras clave break y continue se usan dentro de bucles (for, while, do...while, switch).

Break

Cuando se ejecuta, el bucle termina aunque la condición siga siendo verdadera.

Continue

Sirve para saltar la iteración actual y continuar con la siguiente. No detiene el ciclo, solo se “brinca” el código que queda dentro del bucle.

Funciones

¿Cómo se declara una función y cómo se llama en JavaScript?

En JavaScript, una función es un bloque de código que se puede reutilizar. Se declara (definir) y luego se llama (usar).

declarar una funcion

Hay varias formas, la más común es con la palabra clave function:

function saludar() {
console.log("¡que onda padre!");
}

Llamar una función

Para ejecutarla, escribes su nombre seguido de paréntesis:

saludar();

Función con parámetros

Se le pueden pasar valores:

function sumar(a, b) {
return a + b;
}
console.log(sumar(5, 3));

Función tipo expresión

Se guarda en una variable:

let multiplicar = function(x, y) {
return x * y;
};
console.log(multiplicar(4, 2));

¿Qué significa que una función tenga parámetros y un valor de retorno?

Valor de retorno

Ejemplo:

function saludar(nombre) {
console.log("Hola " + nombre);
}
saludar("Ammy");
saludar("Annie");

Parámetros en una función

Ejemplo:

function sumar(a, b) {
return a + b;
}
let resultado = sumar(4, 6);
console.log(resultado);

¿Qué son las funciones flecha (=>) y qué ventajas tienen?

Las funciones flecha en JavaScript son una manera más corta y moderna de escribir funciones. En lugar de usar la palabra “function”, se escriben con una flechita =>.

Ventajas principales:

1. Son más rápidas de escribir porque no necesitas tanta sintaxis, son ideales cuando la función es pequeña.
2. Tienen un retorno implícito si la función solo hace una operación, no necesitas poner la palabra “return”, ya se entiende automáticamente.
3. No cambian el “this” a diferencia de las funciones normales, mantienen el mismo contexto en el que fueron creadas, lo que evita errores cuando trabajas con objetos o eventos.
4. Se usan mucho en callbacks es decir, en funciones que pasas dentro de otras funciones, como cuando quieres recorrer listas, filtrar datos o transformar valores.

¿Qué es el scope en JavaScript y cómo afecta a las variables?

En JavaScript, el scope es el alcance de una variable, es decir, dónde puede usarse. Puede ser global (se usa en todo el programa), local (solo dentro de una función) o de bloque (solo dentro de llaves { } si se declara con let o const). El scope ayuda a organizar el código y evita errores al usar variables fuera de lugar.

Objetos

¿Cómo se declara un objeto y cómo se accede a sus propiedades?

En JavaScript, un objeto es una estructura que permite guardar información en forma de pares clave: valor.

Declaración de un objeto:

Se hace con llaves { }, donde se escriben las propiedades y sus valores. Ejemplo verbal: un objeto de una persona podría tener nombre, edad y ciudad.

let persona = {
nombre: "Annie",
edad: 25,
ciudad: "México"
};

Acceder a sus propiedades:

Con punto (.)

console.log(persona.nombre);

Con corchetes ([])

console.log(persona["edad"]);

¿Qué diferencia hay entre una propiedad y un método en un objeto?

En JavaScript, dentro de un objeto una propiedad es una característica o dato que describe al objeto, mientras que un método es una función que está dentro del objeto y representa una acción que este puede realizar.

Por ejemplo, si pensamos en un objeto llamado auto, sus propiedades podrían ser el color, la marca y el año, porque describen cómo es el auto. En cambio, sus métodos serían acciones como encender, frenar o acelerar, porque son cosas que el auto puede hacer.

3 métodos básicos de los arrays (push, pop, map, etc.) y explica cómo funcionan.

1. push() agrega un nuevo elemento al final del array.
2. pop() elimina el último elemento del array.
3. map() crea un nuevo array aplicando una función a cada elemento del original, sin modificarlo.

push añade, pop quita y map transforma.

Otros temas

¿Qué es el DOM y por qué es importante?

es una representación en forma de árbol de todo lo que tiene una página web: etiquetas, texto, imágenes, enlaces, etc. Cada elemento de la página se convierte en un nodo dentro de ese árbol, y gracias al DOM, JavaScript puede acceder, modificar o eliminar esos elementos en tiempo real.

¿Qué es un evento en JavaScript

Un evento en JavaScript es una acción o suceso que ocurre en una página web y que el navegador puede detectar, como cuando el usuario hace clic, escribe o mueve el mouse. Los eventos son importantes porque permiten que una página sea interactiva, ya que podemos programar funciones que se ejecuten cuando algo suceda.

Ejemplos comunes de eventos:

1. click. cuando el usuario hace clic en un elemento.
2. mouseover. cuando el puntero del mouse pasa sobre un elemento.
3. keydown → cuando se presiona una tecla del teclado.

¿Cuál es la diferencia entre alert, prompt y console.log?

La diferencia está en cómo muestran información y para qué se usan:

1. alert() Muestra un cuadro emergente con un mensaje y un botón de "Aceptar ". Se usa para dar avisos rápidos al usuario.
2. prompt() Muestra un cuadro emergente que además de un mensaje incluye una caja de texto para que el usuario escriba algo y lo devuelva como respuesta.
3. console.log() → Envía información a la consola del navegador (no al usuario directamente). Se usa sobre todo para probar y depurar el código.

¿Cómo se abre la consola del navegador y qué ventajas tiene usarla para depurar código?

la consola se abre presionando F12 o con la combinación Ctrl + Shift + I (o Cmd + Option + I en Mac) y luego seleccionando la pestaña Console.

La consola permite ver mensajes, errores y resultados de tu código JavaScript en tiempo real. Sus ventajas son:

1. Detectar errores rápidamente y ver exactamente dónde ocurren.
2. Probar fragmentos de código sin tener que recargar la página.
3. Inspeccionar variables, objetos y estructuras de datos mientras el programa se ejecuta.
