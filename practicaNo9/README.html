Descripción General

Sector 404 es una aplicación web para la administración de sistemas médicos. Permite gestionar de manera eficiente pacientes, médicos, especialidades, citas, pagos y servicios en un entorno seguro y fácil de usar.

  • Gestión de usuarios con autenticación
  • CRUD completo de médicos, especialidades, servicios y pagos
  • Control de pacientes y citas
  • Registro de bitácora de acceso
  • Interfaz responsiva con Bootstrap 5
  • Comentarios en español para facilitar aprendizaje
  • Prepared statements para seguridad
  • Diseño oscuro profesional

Tecnologías Utilizadas

Backend

PHP 8.x con arquitectura MVC

Base de Datos

MySQL / MariaDB 10.4+

Frontend

HTML5, CSS3, Bootstrap 5, JavaScript

Módulos CRUD Funcionales

Control de Médicos

Archivo: medicos.php

  • Crear nuevo médico
  • Editar información completa
  • Eliminar médico (borrado lógico)
  • Buscar y filtrar en tiempo real
  • Validación de cédula única
  • Asociar especialidad

Especialidades Médicas

Archivo: especialidades.php

  • Crear nueva especialidad
  • Editar nombre y descripción
  • Eliminar especialidad
  • Listar todas las especialidades

Servicios / Tarifas

Archivo: servicios.php

  • Crear nuevo servicio
  • Editar costo y descripción
  • Eliminar servicio
  • Asociar a especialidad (opcional)

Pagos

Archivo: pagos.php

  • Registrar nuevo pago
  • Editar información del pago
  • Anular pago (cambio de estado)
  • Listar pagos recientes

Instalación y Configuración

Requisitos Previos: PHP 8.0+, MySQL 5.7+, Servidor web (Apache/Nginx), Navegador moderno

Paso 1: Clonar Repositorio

git clone https://github.com/Emm1186/2430117-AW.git
cd 2430117-AW/practicaNo9

Paso 2: Crear Base de Datos

mysql -u root -p
CREATE DATABASE IF NOT EXISTS sector404;
CREATE USER 'sectoruser'@'localhost' IDENTIFIED BY 'TuPasswordFuerteAqui!';
GRANT ALL PRIVILEGES ON sector404.* TO 'sectoruser'@'localhost';
FLUSH PRIVILEGES;
EXIT;

Paso 3: Importar Datos

mysql -u sectoruser -p sector404 < sector404.sql

Paso 4: Verificar Sintaxis

php -l Conexion/conexion.php
php -l Entrada/login.php
php -l dashboard.php

Notas de Seguridad

Advertencia: Las contraseñas están en texto plano para propósitos educativos. NO USAR EN PRODUCCIÓN. Se recomienda implementar bcrypt/password_hash.
  • Usar HTTPS en servidores públicos
  • Implementar hashing de contraseñas (bcrypt)
  • Validar entrada de usuario en todos los formularios
  • Usar prepared statements (ya implementado)
  • Mantener copias de seguridad regulares
  • Revisar logs de acceso periódicamente
